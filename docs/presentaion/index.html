01 · Project Overview

Problem & Solution

Problem Statement
  • Existing LMS platforms have poor UX & are difficult to navigate
  • Low-cost servers crash when running full-stack Next.js + Express apps
  • Raw database & app ports exposed to the internet
  • Manual deployments via SSH are slow & error-prone
Our Solution — SPM LMS
  • Responsive Next.js portal with Teacher Syllabus + Student Progress dashboards
  • Neon Serverless Postgres with Prisma ORM for type-safe queries
  • Nginx Reverse Proxy — single Port 80 entry, internal container routing
  • Automated CI/CD via GitHub Actions with SSH auto-deploy to VPS

06 · DevOps — Git Workflow

Branching Strategy

  • main — stable, production-ready. Protected from direct commits.
  • develop — integration branch for reviewed features.
  • feature/* — isolated dev branches (e.g. feature/auth-jwt)
  • All merges require a Pull Request and code review.
  • Every push to main triggers the CI/CD pipeline automatically.

11 · Challenges & Solutions

What We Overcame

01

VPS Memory Crash

Next.js next build killed by OOM killer on 1GB RAM

Configured 4GB SSD-backed swap space to prevent build crashes
02

Missing Prisma Client

CI pipeline TypeScript build failed — generated files not in source tree

Added npx prisma generate step before every build & test job
03

Docker CORS Errors

Container-to-container requests blocked by cross-origin policy

Standardized Express CORS config & correct NEXT_PUBLIC_API_URL build args
04

Large Docker Images

Initial image builds exceeded 1.5 GB — too large for low-RAM VPS

Multi-stage builds with Alpine runner reduced images to < 250 MB
